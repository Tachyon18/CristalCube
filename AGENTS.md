# AGENTS.md

이 문서는 CristalCube 저장소에서 작업하는 AI 에이전트와 자동화 도구를 위한 저장소 전용 규칙이다.

## 1. 인코딩 안전 규칙

- 이 저장소에는 한글 주석이 들어간 비 UTF-8 소스 파일이 존재한다.
- `file <path>` 등으로 확인했을 때 UTF-8 또는 ASCII가 아닌 텍스트 파일은, 인코딩 변환 승인 없이 수정하지 않는다.
- 비 UTF-8 파일을 UTF-8로 재저장하거나 에디터가 자동 변환하도록 두는 변경도 금지한다.
- 비 UTF-8 파일 수정이 꼭 필요하면 먼저 사용자 승인 또는 유지보수자 승인부터 받는다.
- 인코딩이 의심되는 파일은 내용 변경 전에 현재 인코딩과 예상 영향 범위를 PR 본문에 명시한다.

## 2. 변경 범위 원칙

- 요청된 범위만 최소 수정한다. 관련 없는 리팩터링, 포맷 일괄 변경, 파일 이동은 하지 않는다.
- 새 규칙 문서나 스크립트는 저장소 루트 또는 목적이 분명한 폴더에 추가한다.
- 기존 네이밍 규칙인 `CC_`, `ACC_` 등의 Unreal 클래스 접두어를 유지한다.

## 3. Unreal 프로젝트 취급 규칙

- 주된 텍스트 변경 대상은 `Source/`, `Config/`, `*.uproject`로 제한한다.
- `Content/` 아래 에셋은 Unreal 바이너리 자산일 가능성이 높으므로, 명시적 요청 없이 수정, 삭제, 이동, 리네임하지 않는다.
- 로컬 빌드나 에디터 실행으로 생긴 산출물은 커밋하지 않는다.
- `Binaries/`, `DerivedDataCache/`, `Intermediate/`, `Saved/` 같은 생성 폴더가 생겨도 작업 목적이 아니면 무시한다.

## 4. 작업 절차

- 작업 시작 전 `git status`와 대상 브랜치를 확인한다.
- 새 작업은 가능하면 `origin/main` 기준 새 브랜치 또는 새 worktree에서 진행한다.
- 다른 사람이 만든 변경을 되돌리지 않는다. 충돌하거나 위험하면 먼저 확인한다.

## 5. 검증과 PR

- 코드나 설정을 바꿨다면 가능한 범위에서 빌드, 테스트, 또는 최소 검증 절차를 수행한다.
- 검증을 못 했으면 PR 본문에 이유를 적는다.
- PR에는 최소한 다음 내용을 포함한다.
- 변경 목적
- 핵심 변경 사항
- 검증 여부
- 인코딩 민감 파일 또는 바이너리 에셋을 건드렸는지 여부
